Technologies to be used -
    - Java
    - Maven
    - Hibernate
    - AWS RDS (Postgres)
    - Selenium
    - Cucumber
    - Junit
    - HTML
    - CSS
    - JavaScript
    - Dbeaver

Entities Tracker
    Employees:
    - Id (int) (primary key)
    - Firstname (varchar)
    - Lastname (varchar)
    - Username (varchar)
    - Password (varchar)
    - Role (varchar)

    Requests:
    - Ticketnumber (int) (primary key)
    - Request Description (varchar(500))
    - Money (int) (no more than 1000)
    - Status (varchar) (P/A/R)
    - Manager description (varchar(500))

User stories and associated unit/service tests
   As employee I want to be able to login 
    - Unit Testing 
        - Employee logging in
            - Method: login()
    -Service Testing 
        - Username must be unique
            - Method: checkUsername()

    As an employee I want to be able to view my request
    - Unit Testing
        - Employee view request
            - Method: viewRequest()
    - Service Testing 
        - Employee can only view their own request
            - Method: checkRequest()

    As an manager I want to be able to view all request
    - Unit Testing
        - Manager view request
            - Method: viewAllRequest()
    - Service Testing 
        - N/a

   As an employee I want to create a request
    - Unit Testing
        - Employee create request
            - Method: createRequest()
    - Service Testing 
        - Request cannot exceed $1000
            - Method: moneyLimit()
        - Request cannot exceed 500 character
            - Method: characterLimit()

   As a manager I want to update the status of the request
    - Unit Testing
        - Manager update request status 
            - Method: updateRequest()
    - Service Testing 
        - Request cannot exceed 500 character
            - Method: characterLimit()
